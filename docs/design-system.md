# Art Passport — 디자인 시스템

> 문화예술 예매·아카이빙 앱의 디자인 규칙서.
> 레퍼런스: 애플 월렛 · 실제 여권/보딩패스
> VS Code의 Claude에게 화면을 만들라고 할 때 이 문서를 함께 첨부하면,
> 모든 화면이 아래 규칙에 따라 일관되게 만들어집니다.

---

## 0. 디자인 컨셉 (모든 결정의 근거)

| 컨셉 | 담당 | 한 문장 |
|---|---|---|
| 1. Glanceable | 어떻게 보이나 | 한눈에 읽히는 정보 위계 |
| 2. Stack & Collect | 무엇을 주나 | 쌓고 수집하는 만족감 |
| 3. Living Pass | 왜 특별한가 | 시간에 따라 살아있는 티켓 |

- **Glanceable** -> 타이포 위계를 뚜렷하게, 여백을 넉넉하게.
- **Stack & Collect** -> 카드는 겹쳐도 카테고리 색으로 구분. 스탬프는 수집 단위.
- **Living Pass** -> 티켓은 예매->임박->관람후 상태로 변한다. 카테고리 색은 정식 토큰.

## 1. 색상 (Color)

### 1-1. 기본 팔레트 (Base)

| 토큰 | 역할 | HEX |
|---|---|---|
| `navy` (Primary) | 여권 표지, 헤더, 강조 | `#1B2A4A` |
| `blue` (Secondary) | 버튼, 링크, 강조 텍스트 | `#3D5A8A` |
| `cream` (Background) | 화면 전체 배경 | `#F5F1E8` |
| `surface` | 카드/목록의 흰 배경 | `#FFFFFF` |
| `gold` (Accent) | 로고, 스탬프 테두리, 뱃지 포인트 | `#C9A961` |
| `text-primary` | 본문 진한 글씨 | `#1B2A4A` |
| `text-secondary` | 보조/라벨 회색 글씨 | `#6B6B6B` |
| `text-on-color` | 컬러 배경 위 흰 글씨 | `#FFFFFF` |
| `text-on-color-muted` | 컬러 배경 위 연한 라벨 | `#AEC4F2` (파랑 기준, 카테고리별 조정) |

> 골드는 "특별한 순간"(스탬프, 뱃지, 로고)에만 소량. 남색+크림이 화면의 90%.

### 1-2. 카테고리 색 (Category) — 각 1개

> 보딩패스 배경/헤더에 쓰이는 색. events.genre 값과 1:1 매핑.

| 카테고리 | 토큰 | HEX | 아이콘(outline) |
|---|---|---|---|
| 전시 | `cat-exhibition` | `#1B63C6` | palette |
| 클래식·무용 | `cat-classic` | `#6A5ACD` | music |
| 콘서트 | `cat-concert` | `#7FD4C1` | headphones |
| 연극 | `cat-theater` | `#D97757` | masks-theater |
| 뮤지컬 | `cat-musical` | `#C9599E` | microphone-2 |

> 전시·콘서트는 시안 확정값. 클래식·연극·뮤지컬은 제안값 —
> 피그마에서 조정하면 이 칸만 바꾸세요.

### 1-3. 라이트/다크 모드 (Light / Dark)

앱은 폰의 라이트/다크 모드를 따라간다. 화면마다 두 벌의 색을 준비한다.

| 용도 | 라이트 모드 | 다크 모드 |
|---|---|---|
| 화면 배경 | cream `#F5F1E8` | 다크 `#1A1A1C` |
| 카드/스탬프 빈칸 배경 | 아주 연한 그레이 `#ECEAE3` | 시안 원본 다크 톤 |
| 기본 텍스트 | navy `#1B2A4A` | 밝은 `#F5F1E8` |
| 카테고리 색 | 동일 (양쪽 공통) | 동일 (양쪽 공통) |

> 카테고리 색(전시·콘서트 등)은 두 모드에서 같은 값을 쓴다.
> 배경과 텍스트만 모드에 따라 바뀐다.

## 5. 컴포넌트 (Component)

| 컴포넌트 | 설명 | 상태 |
|---|---|---|
| `BoardingPassCard` | 보딩패스 카드 한 장 (카테고리 색 적용) | 디자인 완료 |
| `BoardingPassStack` | 여러 장 겹쳐 쌓인 월렛 | 디자인 완료 |
| `LabelValue` | 라벨(연한색) + 값(흰색) 세트 | 규칙 확정 |
| `GenreBadge` | 카테고리 아이콘 + 이름 | 규칙 확정 |
| `TicketState` | 예매완료 / 임박 / 관람후 상태 표시 | 컨셉 확정 |
| `Stamp` | 여권 스탬프 1칸 = 공연 포스터 + 관람일 + 장소 | 디자인 완료 |
| `PassportPage` | 스탬프 9칸(3x3) 한 페이지. 안 채운 칸은 점선 | 디자인 완료 |
| `CouponBanner` | 9칸 완성 시 "WE GOT A COUPON!" + 리워드함 버튼 | 디자인 완료 |
| `SearchBar` | 내 티켓 검색바 | 디자인 예정 |
| `EventListCard` | 공연 목록 카드 | 미정 |
| `PrimaryButton` | 기본 버튼 (navy 배경 / 흰 글씨) | 규칙 확정 |

### 스탬프 페이지 규칙 (Stamp Page)
- 한 페이지 = 9칸 (3x3 그리드).
- 스탬프 칸 = 공연/전시 포스터 이미지 + 하단 정보 띠.
- 하단 정보 띠: 날짜는 왼쪽 정렬, 장소는 오른쪽 정렬.
- 콘텐츠(장르) 아이콘은 날짜에 살짝 겹치도록 배치.
- 안 채운 칸: 각 칸마다 점선 테두리 빈 칸으로 표시 (채운 개수만큼만 스탬프, 나머지는 점선 칸). 9칸을 항상 그린다.
- 9칸을 다 채우면 쿠폰 1장 발급 (다음 예매 10% 할인) + 쿠폰 배너 노출.
- 페이지는 계속 다음 장으로 이어짐.
- 페이지 번호(예: 01)는 화면 맨 아래 중앙, 하단 탭 바로 위에 표시.
- 라이트/다크 모드 모두 대응 (1-3 참고).

### 상태별 티켓 시각 규칙 (Living Pass)

| 상태 | 배경 | 아이콘 | 느낌 |
|---|---|---|---|
| 예매완료 | 연회색 `#EBEEF5` | ticket | 조용함 |
| 관람 임박 | 카테고리 색 | plane-departure | 부상, 월렛 맨 앞 |
| 관람 후 | 크림 `#F5F1E8` + 골드 | stamp | 스탬프 아카이빙 |

## 6. 아이콘 (Icon)

- 카테고리 아이콘은 1-2 표 참고.

## 8. 피그마 실측값 (Figma Specs)

> 피그마 시안에서 직접 잰 정확한 값. 이 화면들은 아래 수치를 그대로 따른다.
> 폰트는 전체 Noto Sans KR. 굵기는 각 항목에 표기.

### 8-1. 보딩패스 카드 (Boarding Pass)

**카드 전체**
- 크기: 270 x 380
- 모서리 radius: 10
- 세로 3영역으로 구성: 상단(270x42) / 중간(270x170) / 하단(270x168)

**상단 (270 x 42)**
- 좌: 카테고리명 — Bold 20, `#FFFFFF`
- 우: 카테고리 아이콘 — 높이 24, `#000000` (outline)

**중간 (270 x 161)**
- 라벨류(자택 / 콘텐츠장소 라벨 / PASSENGER / DATE / TIME / SEAT / CAP)
  — Bold 10, `#2C2C2C`
- 비행기 아이콘 — 가로세로 28, `#000000`.
  위치: "자택(출발)" 값과 "콘텐츠 장소 도시(도착)" 값 사이, 세로 중앙 정렬
- 큰 값: HOME(출발), 도착 도시 — DemiLight 24, `#FFFFFF`
- 나머지 값(이름 / 관람일 / 관람시간 / SEAT값 / 인원수) — DemiLight 12, `#FFFFFF`

**하단 (270 x 168)**
- CONTENT 라벨 — Bold 10, `#2C2C2C`
- CONTENT 이름(값) — DemiLight 12, `#FFFFFF`
- 그 아래 QR코드 — 가운데 정렬

> 참고: 하단은 포스터가 아니라 QR코드. (포스터 이미지는 스탬프 페이지에서만 사용)
> 카드 배경색 = 공연 카테고리 색 (1-2 표).

### 8-2. 스탬프 페이지 (Passport Stamp)

**그리드**
- 3 x 3 (한 페이지 9칸)
- 칸 사이 간격: 가로 5, 세로 47 (세로 여백을 넓게 둬 여권 페이지 느낌)

**스탬프 한 칸**
- 크기: 120 x 150, 모서리 radius 5
- 세로 2영역: 상단(120x120) / 하단(120x30)

**상단 (120 x 120)**
- 공연/전시 포스터 이미지

**하단 (120 x 30)** — 위/아래 두 파트로 나뉨
- 위 파트(120x15): 콘텐츠 장소(값) — 우측 정렬, Bold 10, `#2C2C2C`
- 아래 파트(120x15): 관람일자(값) — 좌측 정렬, DemiLight 12, `#FFFFFF`
- 콘텐츠별 아이콘: 하단 전체(120x30) 좌측에 배치, `#000000`, 높이 24.
  보딩패스에서 쓴 카테고리 아이콘과 동일.
  관람일자와 겹치되 아이콘이 뒤(아래 레이어), 관람일자 흰 글씨가 위에 올라옴.

**안 채운 칸**
- 채운 개수만큼만 포스터 스탬프, 나머지는 각 칸을 점선 테두리 빈 칸으로. 항상 9칸을 그림.

**페이지 번호**
- 화면 맨 아래 중앙, 하단 탭 바로 위 (예: 01)
