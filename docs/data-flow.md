# Art Passport — 데이터 연쇄(상태 흐름) 설계

> "예매 → 마이페이지 기록 → 관람 임박 시 티켓(보딩패스) → 관람 후 스탬프 → 취소"
> 이 연쇄가 **시간에 따라 자동으로** 어떻게 흐르는지 확정하는 문서.
> 테이블 칸 정의는 `docs/data-structure.md`, 화면에 뿌릴 예시 콘텐츠는 `docs/content-examples.md` 참고.

---

## 0. 이 문서가 푸는 헷갈림 3가지

1. **"내가 예매한 것"과 "예매할 수 있는 목록"은 다른 데이터다.**
   - `events` = 예매 탭에 뜨는 **공연 카탈로그** (남이 만든 공연 목록).
   - `bookings` = 그 중 **내가 산 것**. 마이페이지·보딩패스·여권은 전부 `bookings`에서 나온다.
   - 지금 코드엔 `events`(dummy-events.ts)만 있고 `bookings`가 없어서 마이페이지·보딩패스·여권이 보여줄 게 없다. → 데모용 `bookings` 세트를 아래 6장에서 확정한다.

2. **상태(예매완료/관람완료)는 "저장"하지 않고 "시간으로 계산"한다.** (아래 2장)
   - 백엔드(크론)가 아직 없으므로, 관람 시각이 지났는지 여부로 상태를 그때그때 판단한다.
   - 딱 하나 예외: **취소만 저장한다** (`isCancelled` 플래그). 취소는 시간으로 알 수 없으니까.

3. **날짜는 "고정 문자열"이 아니라 "오늘로부터 며칠(offset)"로 잡는다.** (아래 1장)
   - 데모가 언제 켜져도 항상 "임박한 티켓", "지난 관람", "예정된 예매"가 보이게 하기 위함.

---

## 1. 시간 기준: 고정 날짜 대신 offset

현재 `dummy-events.ts`는 `showAt: '2026.08.14 19:30'` 같은 **고정 문자열**이다.
데모 관점에서 문제: 오늘이 그 날짜를 지나가면 "임박 티켓"이 영영 안 뜬다.

**설계 결정 — 데이터는 `offsetDays`(오늘 기준 상대일)로 갖고, 화면에 뿌릴 때만 실제 날짜 문자열로 변환한다.**

- `offsetDays: -7` → 오늘로부터 7일 전 (이미 관람함 → 스탬프)
- `offsetDays: 0` → 오늘
- `offsetDays: +2` → 이틀 뒤 (보딩패스로 뜸)
- `offsetDays: +20` → 20일 뒤 (예매완료지만 보딩패스엔 아직 안 뜸)

> 실제 시각(`showAt: timestamp`)은 `오늘 00:00 + offsetDays일 + 시:분`으로 계산.
> 전시처럼 기간이 있는 건 `offsetStart ~ offsetEnd`로 표현(예매 슬롯 날짜는 별도 offset 하나).
> **Supabase 연동 시**: `events.show_at`은 그냥 진짜 timestamp를 쓰면 되고, offset은 데모 더미에서만 쓰는 개념이다.

## 2. 상태를 시간으로 계산하는 규칙 (가장 중요)

`bookings`의 상태는 저장하지 않고 아래 순서로 **매번 계산**한다. (`now` = 현재 시각, `showAt` = 관람 시각)

```
if (booking.isCancelled)      → 상태 = '취소'
else if (now < showAt)        → 상태 = '예매완료'
else (now >= showAt)          → 상태 = '관람완료'   // 이 순간 스탬프가 존재하게 됨
```

여기서 파생되는 4가지 판정:

| 판정 | 조건 |
|---|---|
| **보딩패스에 뜬다** | 상태='예매완료' **그리고** 관람일이 "오늘 포함 3일 이내"(달력 기준, `now < showAt`) |
| **예매 예정(대기)** | 상태='예매완료' **그리고** 관람일이 3일보다 더 남음 (아직 보딩패스엔 안 뜸) |
| **스탬프가 있다** | 상태='관람완료' (즉 취소 아님 + 관람 시각 지남) |
| **취소됨** | `isCancelled = true` |

> **핵심**: 관람 시각을 지나는 그 순간, 같은 booking이 자동으로
> "보딩패스에서 사라지고" + "스탬프로 여권에 찍히고" + "마이페이지에서 관람완료로 이동"한다.
> 이 3가지가 전부 위 계산식 하나에서 나온다. 별도 이벤트/버튼이 필요 없다.

Supabase 연동 후에는 이 계산을 크론(예약 작업)이 대신 돌려 `status`/`watched_at`을 실제로 써넣을 수 있지만,
**프론트 데모 단계에선 화면을 열 때 위 식으로 즉석 계산**하면 된다. (백엔드 없이도 연쇄가 작동)

## 3. 예매 하나의 생애주기 (타임라인)

`레베카(뮤지컬)`를 하나 예매했다고 하자. 이 booking 한 건이 시간에 따라 이렇게 흐른다.

```
[예매하는 순간]
  bookings에 1건 생성 (isCancelled=false)
  → 상태 '예매완료'
  → 마이페이지 "예매완료" 목록에 등장
  → 보딩패스엔 아직 안 뜸 (관람일 3일보다 더 남음)

        │  (시간 흐름)
        ▼
[관람 3일 전이 됨]
  → 상태 여전히 '예매완료'
  → 보딩패스 탭/홈에 뮤지컬 색(#C9599E) 카드로 등장 (월렛 스택에 합류)

        │
        ▼
[관람 시각을 지남]
  → 상태 '관람완료'로 자동 전환
  → 보딩패스에서 사라짐
  → stamps에 도장 1개 생성 → 여권 페이지에 '레베카' 포스터가 도장으로 찍힘
  → 마이페이지 "관람완료" 목록으로 이동

        │
        ▼
[이 도장이 9의 배수 번째였다면]
  → coupons에 '사용가능' 쿠폰 1장 발급 ("다음 예매 10% 할인")
  → 마이페이지 쿠폰함에 등장
```

**취소 분기** (관람 전에만 가능):

```
[예매완료 / 보딩패스 상태에서 '취소' 누름]
  → isCancelled = true
  → 상태 '취소'
  → 보딩패스·"예매완료" 목록에서 즉시 제거
  → 스탬프 안 생김 (여권에 영향 없음)
  → 마이페이지 "취소" 목록에 남음 (기록은 유지)
```

**취소 규칙**: `now < showAt` 일 때만 취소 버튼 노출. **관람완료(스탬프 찍힌 것)는 취소 불가.**

## 4. 각 화면이 무엇을 보여주는가 (데이터 → 화면 매핑)

| 화면 | 무슨 데이터 | 필터 |
|---|---|---|
| **예매 탭 (목록)** | `events` 카탈로그 | 장르로 필터. (권장: `offsetEnd ≥ 0`인 예매 가능한 것만) |
| **예매 상세 `[id]`** | `events` 1건 + 내 쿠폰 | 가격 표시 시 사용가능 쿠폰 있으면 10% 할인가 병기 |
| **보딩패스 (홈/월렛)** | `bookings` | 상태='예매완료' **AND** `showAt-3일 ≤ now < showAt`. 여러 개면 카테고리 색으로 스택 |
| **여권 (스탬프)** | `stamps` (=관람완료 bookings) | `stamped_at`(=showAt) 순 정렬, 9칸씩 페이지. 빈칸은 점선 |
| **마이페이지** | `bookings` 전체 + `coupons` | 아래 5장 참고 |

---

## 5. 마이페이지 구성 (데이터 허브)

마이페이지는 "내 모든 booking + 쿠폰"을 상태별로 모아 보여주는 곳. 제안 구성:

1. **상단 요약**: 닉네임 / 총 관람 수(=스탬프 수) / 모은 쿠폰 수 / 다음 쿠폰까지 남은 칸(`9 - (스탬프수 % 9)`).
2. **예매 내역** — 상태 탭 3개:
   - `예매완료` (예정 + 임박 전부): `now < showAt` & 취소 아님
   - `관람완료`: `now ≥ showAt` & 취소 아님 (여기 항목들이 곧 스탬프)
   - `취소`: `isCancelled`
3. **쿠폰함**: `coupons`를 상태별(사용가능 / 사용완료 / 만료)로.

> 여권 탭과 마이페이지 "관람완료"는 **같은 원천(관람 완료된 bookings)**을 다르게 보여주는 것뿐이다.
> 여권 = 포스터를 도장 그리드로, 마이페이지 = 목록으로.

## 6. 데모용 bookings 세트 (확정)

모든 상태가 화면에 보이도록 아래처럼 18건을 짜둔다. `offsetDays`는 오늘 기준.
(각 항목의 공연 상세·가격은 `docs/content-examples.md`의 카탈로그에서 가져옴)

### 6-1. 관람완료 → 스탬프 (10건) — 여권 채우기용
`offsetDays`가 음수 = 이미 지남. 오래된 순서대로 스탬프가 찍힌다.

| # | 공연 | 장르 | offsetDays | 결과 |
|---|---|---|---|---|
| 1 | 요시고 사진전 | 전시 | -60 | 스탬프 1 |
| 2 | 서울시향 말러 교향곡 5번 | 클래식·무용 | -52 | 스탬프 2 |
| 3 | 검정치마 라이브 | 콘서트 | -45 | 스탬프 3 |
| 4 | 고도를 기다리며 | 연극 | -40 | 스탬프 4 |
| 5 | 지킬 앤 하이드 | 뮤지컬 | -33 | 스탬프 5 |
| 6 | 데이비드 호크니 | 전시 | -26 | 스탬프 6 |
| 7 | 조성진 피아노 리사이틀 | 클래식·무용 | -20 | 스탬프 7 |
| 8 | 십센치(10cm) 콘서트 | 콘서트 | -14 | 스탬프 8 |
| 9 | 인간 실격 | 연극 | -7 | 스탬프 9 → **쿠폰 1장 발급** |
| 10 | 오페라의 유령 | 뮤지컬 | -3 | 스탬프 10 → 2페이지 1칸째 |

→ 여권: **1페이지 꽉 참(9칸) + 2페이지 1칸**. 남은 8칸은 점선 빈칸으로 표시. "다음 쿠폰까지 8칸".

### 6-2. 보딩패스 (3건) — 관람 임박, 색 겹침용
`0 ≤ offsetDays ≤ 3` 이라 월렛 스택에 뜬다. 카테고리 색이 서로 달라 스택 구분이 보인다.

| 공연 | 장르 | offsetDays | 색 |
|---|---|---|---|
| 백조의 호수 | 클래식·무용 | +1 | #6A5ACD |
| 아이유 콘서트 'The Golden Hour' | 콘서트 | +2 | #7FD4C1 |
| 햄릿 | 연극 | +3 | #D97757 |

### 6-3. 예매완료(예정) (3건) — 아직 보딩패스 안 뜸
`offsetDays > 3`. 마이페이지 "예매완료"엔 있지만 보딩패스 스택엔 없음.

| 공연 | 장르 | offsetDays |
|---|---|---|
| 레베카 | 뮤지컬 | +12 |
| 반 고흐: 별이 빛나는 밤 | 전시 | +20 |
| 위키드 | 뮤지컬 | +30 |

### 6-4. 취소 (2건)
관람 전에 취소한 것. `isCancelled=true`. 스탬프·보딩패스엔 없고 마이페이지 "취소"에만 남음.

| 공연 | 장르 | 원래 offsetDays | 상태 |
|---|---|---|---|
| 자우림 25주년 콘서트 | 콘서트 | +8 | 취소 |
| 지젤 | 클래식·무용 | +6 | 취소 |

### 6-5. 쿠폰 (파생 + 데모)
- 6-1의 9번째 스탬프에서 **자동 발급된 1장** → 상태 `사용가능`.
- (선택) 데모 다양성용으로 과거에 쓴 `사용완료` 1장을 수동으로 추가해도 됨.

## 7. 요약 상태 전이도

```
                 예매하기
                    │
                    ▼
              ┌───────────┐   now < showAt-3d
              │  예매완료   │───────────────► 마이페이지 "예매완료"에만
              └───────────┘
                 │      │
    showAt-3d≤now│      │ 취소(관람 전만)
        <showAt  │      ▼
                 │  ┌────────┐
                 │  │  취소   │──► 마이페이지 "취소" (기록 유지, 스탬프X)
                 ▼  └────────┘
           ┌───────────┐
           │ 보딩패스 노출 │ (상태는 여전히 예매완료)
           └───────────┘
                 │ now ≥ showAt
                 ▼
           ┌───────────┐
           │  관람완료   │──► 스탬프 1개 + 여권에 포스터 찍힘
           └───────────┘        └─ 9의 배수째면 쿠폰 1장 발급
```

## 8. 다음 작업으로 넘어갈 때 체크리스트

이 설계를 코드로 옮길 때(추후, 별도 요청 시):
1. `dummy-events.ts`를 `offsetDays` 기반으로 확장 (`content-examples.md`의 50개).
2. `data/dummy-bookings.ts` 신설 — 위 6장 18건 (`eventId` + `offsetDays` + `isCancelled`).
3. 파생 헬퍼 1개 — `booking + now`를 받아 `{ status, showAt, isBoardingPass, hasStamp }` 계산 (2장 규칙).
4. 스탬프·쿠폰은 저장하지 않고 관람완료 bookings에서 파생 계산 (여권/쿠폰 화면에서).
