# Art Passport — 프로젝트 지침서

이 파일은 이 프로젝트에서 작업할 때 반드시 먼저 읽고 따라야 할 규칙입니다.

## 프로젝트 개요
- 문화예술(전시·클래식/무용·콘서트·연극·뮤지컬) 예매 및 아카이빙 모바일 앱.
- 핵심 컨셉: 예매로 끝나지 않고, 티켓을 보딩패스로 보여주고 관람 후 여권 스탬프로 아카이빙.
- 레퍼런스: 애플 월렛, 실제 여권/보딩패스.

## 기술 스택
- 프론트엔드: React Native + Expo (SDK 54)
- 언어: TypeScript
- 백엔드/DB/인증: Supabase (아직 미연동 — 프론트 먼저 개발 중)
- 결제: 테스트 결제만 (데모/포트폴리오용)

## 디자인 원칙 (design-system.md 요약)
- 3대 컨셉: Glanceable(한눈에 읽힘) / Stack & Collect(쌓고 수집) / Living Pass(상태 변화).
- 기본색: navy #1B2A4A, cream #F5F1E8(배경), gold #C9A961(포인트, 소량만).
- 카테고리색: 전시 #2B5FD9, 클래식·무용 #6A5ACD, 콘서트 #7FD4C1, 연극 #D97757, 뮤지컬 #C9599E.
- 아이콘은 실선(outline)으로 통일.
- 카드 radius 16px, 여백은 4의 배수.

## 핵심 기능 규칙 (data-structure.md 요약)
- 여권 한 페이지 = 스탬프 9칸(3x3). 페이지는 계속 다음 장으로 이어짐.
- 스탬프 칸에는 공연/전시 포스터 이미지가 들어감 (포스터 = 스탬프).
- 안 채운 칸은 점선 테두리로 표시.
- 스탬프 9개를 채울 때마다 쿠폰 1장 발급 = "다음 예매 10% 할인".
- 관람 시간이 지나면 해당 예매가 자동으로 스탬프가 됨.
- 보딩패스: 관람일 임박한 예매를 월렛처럼 표시. 여러 개면 겹쳐 쌓음(카테고리 색 구분).
- 예매 상태: 예매완료 / 관람완료 / 취소. 쿠폰 상태: 사용가능 / 사용완료 / 만료.

## 작업 규칙
- 한 번에 하나의 화면/컴포넌트만 만든다. 여러 개를 동시에 만들지 않는다.
- 새 기능은 별도 브랜치에서 작업한다 (예: feature/stamp-page).
- 정지된 화면(색·레이아웃)을 먼저 완성하고, 애니메이션은 그 다음 단계에서 추가한다.
- 아직 실제 데이터가 없으므로, 화면은 예시(더미) 데이터로 먼저 만든다.
- 코드는 초심자도 이해할 수 있게, 복잡한 부분엔 주석으로 설명을 단다.

## 하지 말 것
- 요청하지 않은 화면이나 기능을 임의로 추가하지 않는다.
- 디자인 시스템에 없는 색·크기를 임의로 쓰지 않는다.
